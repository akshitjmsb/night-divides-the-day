## 1. Brand Overview

### Brand Name
**Night Divides the Day**

### Brand Concept
A personal dashboard application that adapts to the rhythm of day and night. The app operates in two distinct modes based on time of day, creating a unique experience that reflects the natural division between day and night.

### Core Philosophy
- **Time-Aware**: The application adapts its behavior and content based on canonical time (6 AM - 6 PM = Day Mode, 6 PM - 6 AM = Night Mode)
- **Reflective**: Encourages daily philosophical reflection through curated quotes
- **Personal**: A private, personal dashboard for learning, tasks, and self-improvement
- **Minimalist**: Clean, uncluttered interface that focuses on content
- **Mobile-First**: Designed for mobile devices with responsive scaling to desktop

### Brand Values
1. **Simplicity** - Clean, uncluttered interfaces
2. **Thoughtfulness** - Encourages reflection and intentionality
3. **Consistency** - Predictable, reliable experience across all features
4. **Accessibility** - Touch-friendly, readable, and accessible to all users
5. **Elegance** - Subtle, refined design that doesn't distract from content

## 7. Responsive Design

### Breakpoints
- **Mobile**: `max-width: 640px` (sm)
- **Tablet**: `641px - 1024px`
- **Desktop**: `min-width: 1025px`

### Mobile-First Principles
1. **Touch Targets**: Minimum 44px height/width
2. **Spacing**: Larger padding on mobile (0.75rem+)
3. **Typography**: Slightly larger on mobile for readability
4. **Grid**: 3 columns mobile, scales up to 6 on desktop
5. **Navigation**: Vertical icon+text layout on mobile

### Responsive Patterns
- **Cards**: Full width on mobile, constrained on desktop
- **Modals**: Full viewport on mobile, centered on desktop
- **Forms**: Stack vertically on mobile, horizontal on desktop
- **Text**: Scales down on mobile, larger on desktop

## 8. Shadows & Depth

### Shadow Scale
- **Subtle**: `0 1px 3px rgba(0, 0, 0, 0.05)`
- **Standard**: `0 2px 4px -1px rgba(0, 0, 0, 0.1), 0 1px 2px -1px rgba(0, 0, 0, 0.06)`
- **Hover**: `0 4px 8px -2px rgba(0, 0, 0, 0.15), 0 2px 4px -1px rgba(0, 0, 0, 0.1)`
- **Pressed**: `0 2px 4px -1px rgba(0, 0, 0, 0.3)`

### Usage Guidelines
- Use subtle shadows for cards and containers
- Increase shadow on hover for interactive elements
- Reduce shadow on active/pressed states
- Avoid heavy shadows - keep it minimal

## 9. Animations & Transitions

### Transition Timing
- **Standard**: `0.2s ease`
- **Smooth**: `0.3s ease`
- **Fast**: `0.15s ease`

### Common Animations
- **Hover Lift**: `translateY(-1px)` with shadow increase
- **Active Press**: `translateY(0)` with shadow decrease
- **Scale**: `scale(0.98)` for button presses
- **Fade**: `opacity` transitions for show/hide
- **Slide**: `transform` transitions for modals

### Performance
- Use `transform` and `opacity` for animations (GPU-accelerated)
- Avoid animating `width`, `height`, `top`, `left`
- Use `will-change` sparingly and only when needed

## 10. Accessibility

### Touch Targets
- **Minimum Size**: 44px √ó 44px
- **Spacing**: At least 8px between touch targets
- **Padding**: Generous padding for easier tapping

### Focus States
- **Outline**: `2px solid #3b82f6` with `2px` offset
- **Glow**: `0 0 0 3px rgba(59, 130, 246, 0.1)` box-shadow
- **Always Visible**: Never remove focus indicators

### Color Contrast
- **Text on Background**: Minimum 4.5:1 ratio
- **Large Text**: Minimum 3:1 ratio
- **Interactive Elements**: High contrast for visibility

### Semantic HTML
- Use proper heading hierarchy (h1 ‚Üí h2 ‚Üí h3)
- Use `aria-label` for icon-only buttons
- Use `role` attributes where appropriate
- Provide `title` attributes for tooltips
